# Регистрация студента — флоу и нюансы

## Нормальный сценарий
1. Студент пишет первое сообщение в группе
2. Бот сохраняет сообщение как `pending_text`, отправляет приветствие + просьбу написать имя
3. Следующее сообщение → `ai.extract_name(text)`
4. Если None → fallback: `ai.extract_name(sender_name)` (Telegram-имя)
5. Если снова None → `ask_name_again`
6. Имя получено → `find_unlinked_by_name` → привязываем к существующей записи или создаём новую
7. Если первое сообщение было отчётом — засчитывается автоматически

## Авто-матч по Telegram-имени (при ПЕРВОМ сообщении)
- Если `sender_name` → `extract_name(sender_name)` → имя валидно
- Ищем в группе точное совпадение (`LOWER` match) с незарегистрированным студентом
- **Только если совпадение ровно одно** (`exact_count == 1`) → авто-регистрация без вопросов
- Если два и более "Азиза" — безопасно пропускаем, спрашиваем имя

## Защита от "НЕТ ·" и подобных
- `extract_name` проверяет первое слово результата (не всю строку)
- Если первое слово = НЕТ/NET/NO/NONE → возвращает None
- Стриппинг `·—-.` из слова перед проверкой
