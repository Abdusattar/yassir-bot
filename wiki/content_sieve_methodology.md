# Методология сита для контентных баз (хадисы, аяты, книги)

> Разработано: 26.06.2026 в ходе построения базы мотивационного контента для Ясир-бота.
> Принцип: **позитивная куратура, не исключение плохого**.

---

## Философия

Не "что исключить" — а "что включить". Как учитель выбирает какие аяты повесить на стену класса.

Нет плохих аятов или хадисов — есть контент, который подходит или не подходит для конкретной педагогической цели. Это важно сформулировать правильно с самого начала, чтобы не попасть в ловушку "исключения плохого".

---

## Шаг 1 — Определить педагогическую цель

Сформулировать один вопрос-критерий. Например:

> «Насколько прямой смысл этого текста побуждает студента к изучению Корана?»

Этот вопрос становится основой для всех последующих шагов.

---

## Шаг 2 — Первичный фильтр по тематическим тегам

Если в базе есть теги — используем их как первое грубое сито.

**Для аятов (9 тегов системы):**

Делим теги на два уровня:

| Уровень | Теги | Логика |
|---|---|---|
| Сильные (всегда проходят) | `quran`, `knowledge`, `striving`, `patience`, `reward`, `remembrance` | Прямая связь с целью |
| Широкие (только с сильным) | `faith`, `mercy`, `character` | Сами по себе слишком широкие |

SQL-паттерн:
```sql
WHERE topic_tags LIKE '%quran%' OR topic_tags LIKE '%knowledge%' OR ...
```

Результат: из ~6200 аятов → 2188 кандидатов.

---

## Шаг 3 — Скоринг 1–5 (правила, без AI)

Для каждой единицы контента считаем балл на основе **правил**, не модели. Это быстро, бесплатно и воспроизводимо.

### Аяты — два источника сигнала:

**Сигнал 1: арабский текст** (паттерны):
```python
# Score +3 (прямая связь):
اقرأ  (Iqra)
القرآن  (слово Коран в тексте)
يتلو / تلاوة  (рецитировать)
طلب العلم  (поиск знания)

# Score +2 (сильная поддержка):
العلم  (знание)
يرفع الله  (Аллах возвышает)
فاصبر / اصبروا  (команда: терпи)
وجاهدوا  (усердствуй)

# Score +1 (полезный фон):
ذكر الله  (зикр)
أجر / أجورهم  (награда)
الصابرين  (терпеливые)
```

**Сигнал 2: теги**:
```python
TAG_WEIGHTS = {
    "quran": 3, "knowledge": 2, "striving": 2,
    "patience": 2, "reward": 1, "remembrance": 1
}
# Бонус +1 если knowledge + (striving или patience)
```

Итоговый балл = arabic_score + tag_score → нормализация в 1–5.

### Хадисы — английский текст:

```python
# Score +3 (прямая связь):
"qur'an", "memoriz", "hifz", "hafiz"
"seek.*knowledge", "best.*quran"

# Score +2 (сильная поддержка):
"knowledge", "patience", "strive", "effort"
"night prayer", "scholar", "best among you"

# Score +1 (полезный фон):
"reward", "paradise", "faith", "dhikr", "sincere"
```

## Шаг 4 — Порог включения

Выбираем порог **score ≥ 3** для включения в рабочий пул сэмплера.

Итоги этой сессии:
- Аяты: 2188 кандидатов → **493 в пуле** (score ≥ 3)
- Хадисы: 3800 кандидатов → **293 в пуле** (score ≥ 3)

Оба пула — годы контента с `used_at`-ротацией.

---

## Шаг 5 — Проверка на известных примерах

Перед применением всегда проверяем несколько конкретных случаев:

```python
# Аят 4:107 [character,mercy] → ❌ не проходит (ожидаемо)
# Аят 20:114 [quran,knowledge,remembrance] → ✅ проходит
# Аят 57:1 [remembrance] → ✅ проходит
# Аят 96:1 [quran] → ✅ проходит (Iqra)
```

## Схема данных (SQLite)

```sql
-- Аяты
ALTER TABLE quran_ayahs ADD COLUMN motiv_score INTEGER;  -- 1-5
-- Хадисы
ALTER TABLE hadiths ADD COLUMN motiv_score INTEGER;  -- 1-5
```

Сэмплер фильтрует:
```sql
WHERE (topic_tags LIKE '%quran%' OR ...)
  AND (motiv_score IS NULL OR motiv_score >= 3)
```

`motiv_score IS NULL` — фолбэк для новых записей до пересчёта.

## Применение к новым книгам / источникам

Для любой новой базы (тафсир, книга по фикху, сборник ду'а и т.д.):

1. **Определи цель** — один вопрос-критерий
2. **Добавь теги** если их нет — хотя бы 5-6 тематических
3. **Адаптируй паттерны** под язык текста (арабский / английский / другой)
4. **Запусти скоринг** — `scripts/score_*.py` как шаблон
5. **Выбери порог** — обычно ≥ 3, можно ≥ 4 для более строгого пула
6. **Проверь на 10-20 известных примерах** вручную

Скрипты-шаблоны: `scripts/score_ayah_motiv.py`, `scripts/score_hadith_motiv.py`

---

## Ключевые принципы

1. **Изобилие → точность**: больший пул позволяет быть строгим без потери покрытия
2. **Позитивный отбор**: спрашиваем "что подходит", не "что исключить"
3. **Правила быстрее AI**: для начального скоринга правила дешевле и воспроизводимее
4. **Порог настраиваемый**: `_AYAH_SCORE_MIN = 3` — одна строка для изменения всего пула
5. **Теги + текст**: двойной сигнал надёжнее одного источника
