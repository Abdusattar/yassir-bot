# Детекция отчётов (check_text)

## Принцип

`check_text(text, media=False)` в `core/db.py` — keyword-based парсинг, без AI.  
Возвращает `{m, r, t, j, n, h: bool}` по TASK_WORDS из `core/content.py`.

## Особые правила для задания `t` (муфрадат/слова)

Задание `t` засчитывается при выполнении ЛЮБОГО из условий:

1. **Ключевое слово + арабский** — слово из TASK_WORDS["t"] ("слова", "перевод", "муфрадат", ...) И `_has_arabic(text)`.
2. **Словарные пары без ключевого слова** — regex `[arabic]{2,} - \S` встречается 2+ раз.  
   Порог 2+ критичен: хадис/аят — одна цитата (1 пара), словарный список — много пар.
3. **Медиа (фото с caption)** — для `media=True` арабский не требуется (студент сфотографировал написанное).

Отрицания блокируют засчёт: "не буду сдавать переводы" → `t=False`.

## Фото без caption

Фото с пустым caption (`not text`) → бот отправляет подсказку через `photo_needs_caption` (i18n).  
**Не засчитывается молча** — студент получает сообщение с legend-ом задач группы и просьбой прислать снова с caption.

## Мерж отчётов (save_report)

`save_report` использует `ON CONFLICT DO UPDATE SET m=MAX(m, excluded.m)`.  
Повторное сообщение в тот же день: задание уже `1` → остаётся `1`, новые задания добавляются.

В handlers.py перед `save_report`: читается `prev = get_today_report()`, prev-задания принудительно сливаются в `tasks_done`. Затем вычисляется `new_tasks = now_done - prev_done` — только новые задания упоминаются в ответе.

## Сценарии второго сообщения

| Сообщение 2 | Поведение |
|---|---|
| Пары `أرض - земля` × 2+ | `t=True` → засчитывается (new_tasks пусто если t уже было) → "уже засчитано" + AI-проверка качества |
| Ключевое слово + арабский | Тот же путь, что и первое сообщение |
| Фото без caption | Подсказка с legend |
| Фото с caption | `media=True`, arabic не требуется для `t` |
