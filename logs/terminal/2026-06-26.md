# Сессия 26.06.2026 (утро)

## Задача
Фикс: студент шлёт отчёт двумя сообщениями — второе не засчитывается.

## Диагноз
- `save_report` использует `MAX()` — мерж, не перезапись. БД не виновата.
- `check_text` требует ключевое слово ("слова", "перевод" и т.д.) + арабский символ для задания `t`.
- Второе сообщение с чистыми парами (`سماء - небо`) без заголовка → `score=0` → тихий return.
- Фото без caption → `if not text: return` → тоже тихо игнорируется.

## Решения

### 1. Авто-определение словарных пар (`core/db.py`)
Добавлен regex: если 2+ паттернов `[arabic]{2,} - [перевод]` и не media → `t=True` без ключевого слова.
Порог 2+ — защита от хадисов/аятов (одна цитата — одна пара, не словарь).

### 2. Фото без caption (`core/handlers.py`)
Вместо тихого return — подсказка студенту: "Пришли снова с caption — что прописывал + legend".

### 3. Новый i18n-ключ (`core/i18n.py`)
`photo_needs_caption` — на 8 языках (ru/ky/uz/kk/tr/ar/en/zh).
