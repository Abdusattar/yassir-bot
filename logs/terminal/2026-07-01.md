## Переработка утреннего отчёта в ТДБР — агрегация по всем группам

Было: отдельное сообщение на каждую группу (10+ сообщений). Стало: одно агрегированное сообщение.

Формат (согласован по шагам с пользователем):
- Шапка — AI-строка (pro-модель) на основе аята/хадиса, тема: почёт стремящихся к знанию Корана
- Блок "Сдали ВСЕ задания за {дата}" — по группам, только полные сдачи, вместо галочек — 🔥страйк (дней подряд, якорь на "вчера", не на "сегодня")
- Частично сдавшие — не показываются вообще (только полные + сводная статистика группы)
- Группа всегда показывает заголовок + строку "📊 X/Y · средний"
- Группы с 0 сдавших — не разворачиваются блоком, а собираются одной строкой внизу: "Не сдавали вчера ничего: G-3, ..."
- Группа, где 0 сдавших целиком объясняется узром — не попадает в этот список
- Отдельная строка "⛔ Узр: Имя (Группа), ..." — глобально по всем группам

Реализация:
- `db.py`: новая `get_daily_task_counts(group_id, group_tasks, for_date)` → [{id,name,done,excused}]
- `db.py`: `get_today_avg` и `get_streak_days` получили параметр даты (`for_date`) — раньше `get_streak_days` всегда анкорился на "сегодня", из-за чего в 07:00 стрик всегда показывал бы 0
- `ai.py`: новая `morning_report_intro(hadith, ayah)` — короткая AI-строка на pro-модели
- `scheduler.py`: `morning_tadabbur_report()` переписан на агрегацию

## Найденные попутно баги (не связаны с задачей, но всплыли)

1. **`s.get("phone")` на `sqlite3.Row`** в двух местах `scheduler.py` (строки ~123, ~223) — `.get()` не существует у Row, падало исключением, которое глушилось внешним `except`. Из-за этого фича "личная насыха несдавшим вчера" (добавлена сегодня утром) не работала ни разу. Исправлено на `s["phone"]`.
2. **Обрыв текста у ТДБР-контента (насыха/тадаббур-пост)** — сегодня утром `_NASIHA_MODEL` переключили с `z-ai/glm-5.2` (вчерашняя модель) на `deepseek/deepseek-v4-pro`. Pro-модель — reasoning-модель, тратит токены на "невидимые" рассуждения ДО видимого текста, из-за чего `max_tokens=900` (и 150 для новой intro-фразы) стабильно не хватало — reproduced на реальных вызовах API (2/3 и 4/5 обрывов). Исправлено: `max_tokens` поднят до 1600 для `daily_tadabbur_post`, `daily_nasiha`, `morning_report_intro`. Проверено 14 живых вызовов подряд — 0 обрывов.
